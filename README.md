## 📖 项目简介

**泡泡互换**是一款基于 **SpringBoot + Vue + UniApp** 的微信小程序，以"盲盒交换"为核心玩法，融合了**社交社区、AI 智能匹配、信用评价体系、实时聊天、拼豆图纸生成**等功能模块。

用户发布自己的盲盒玩具信息（隐藏部分内容制造惊喜感），系统智能匹配交换对象，双方确认后进入发货-收货-完成的全流程。辅以社区动态、关注体系、信用评分，构建了一个完整的轻娱乐交换生态。

> 💡 **创新亮点**：不止是交换工具，还内置了"拍照生成拼豆图纸"的 AI 图像处理引擎——上传任意照片一键生成像素级拼豆图纸，自带色号标注 + 材料清单，可在小红书/抖音做内容引流。

## ✨ 核心功能

### 🔐 用户体系
| 功能 | 说明 |
|------|------|
| 微信授权登录 | 微信 `wx.login()` 静默登录，自动获取 openid + 头像昵称 |
| 实名认证 | 接入**腾讯云 FaceID**，支持微信小程序人脸核身，真实身份认证 |
| 手机绑定 | 微信手机号快速验证 + 加密存储 |
| 个人主页 | 展示交换记录、信用分、背包道具、动态统计 |
| 用户状态管理 | 后台可启用/禁用，管理员账号体系独立 |

### 🎁 盲盒交换核心
| 功能 | 说明 |
|------|------|
| 发布交换贴 | "我有的" + "我想要的"，支持多图上传、成色描述、交换条件 |
| AI 智能匹配 | 根据用户拥有的/想要的盲盒，智能推荐最佳交换对象 |
| 公域信息流 | 首页推荐流 + 热门推荐（加权算法：浏览量+发布时间衰减） |
| 全生命周期状态机 | `PENDING → AGREED → TRANSFERRING → COMPLETED`，含超时/取消/拒绝分支 |
| 行级锁并发控制 | 交换确认采用 MySQL 行锁，防止并发冲突 |
| 交换审计日志 | 每一步操作记录到 `swap_request_audit`，全链路可追溯 |
| 微信订阅消息 | 交换进度变动自动推送微信通知（申请/同意/完成） |

### ⭐ 信用评分体系
| 功能 | 说明 |
|------|------|
| 5.0 初始分（满分 10） | 新用户默认信用分 5.0 |
| 行为加减分 | 发帖 +0.1、申请交换 +0.1、同意交换 +0.2、完成交换 +1.0、被拒 -0.2、超时 -0.5 |
| 低分优先发货 | 信用分低的用户先发货，降低违约风险 |
| 完整审计日志 | `user_credit_log` 记录每次变动，含原因、关联订单号、变动前后余额 |
| 后台手动调整 | 管理后台支持人工加减信用分 |

### 🎨 拼豆图纸生成引擎（独家功能 🆕）
| 功能 | 说明 |
|------|------|
| 照片→像素图纸 | 上传任意照片，一键生成拼豆像素图纸 |
| 多尺寸支持 | 29 / 48 / 58 / 80 / 100 / 120 格，六档可选 |
| 多品牌适配 | MARD / 漫漫 / COCO / 盼盼 / 咪小窝，5 大品牌色卡独立匹配 |
| Oklab 专业色彩引擎 | 采用国际设计标准 Oklab 色彩空间，感知均匀，配色比肉眼更准 |
| 超采样降噪 | 每颗豆子采样原图数百~上千像素取最优颜色，细节保留到位 |
| 色号标注 | 每个格子上标注色号（C1、D2、M3 等），照着买豆子就行 |
| 材料清单 | 自动统计每种颜色需要多少颗豆子，下单直接抄 |
| 历史管理 | 生成的图纸自动保存，支持查看/删除 |

### 技术选型清单

| 层级 | 技术 | 版本/说明 |

| **缓存** | Redis (Lettuce) | 7.x，Pub/Sub + 限流 + Token 缓存 |
| **鉴权** | JWT (jjwt) | HMAC-SHA256，用户 + 管理员双体系 |
| **接口文档** | Knife4j | 4.4.0（OpenAPI 2 规范） |

| **实名认证** | 腾讯云 FaceID | 人脸核身 |
| **消息推送** | 微信订阅消息 | 交换进度通知 |

| **工具库** | Lombok / Hutool / Commons-lang3 | 代码简化 |
| **前端** | UniApp (小程序端) + Vue (管理后台) | 跨平台 / 前后端分离 |

### 关键设计

- **分层架构**：Controller → Service(接口) → ServiceImpl → Mapper → XML，符合阿里 Java 开发手册
- **全局异常处理**：`BizException` + `@RestControllerAdvice`，统一异常拦截与响应格式
- **AOP 日志**：自定义 `@ApiLog` 注解，自动记录接口调用日志
- **ThreadLocal 上下文**：`UserContext` 持有当前请求用户信息，Service 层无需传参
- **JWT 拦截器**：`AuthInterceptor` 拦截所有 `/api/**`，区分公开接口/用户接口/管理员接口
- **行级锁**：交换确认时 `SELECT ... FOR UPDATE`，保障并发安全
- **每日限流**：`user_daily_action` 表 + Redis 双重限流，卡片可扩展上限
- **状态机流转**：交换全生命周期状态机，含超时自动处理
